# API Bingbot Diagnostic Specification

## Problem Description
The server logs show `403 Forbidden` errors specifically for `bingbot` when requesting product data via the API.

## Findings

### 1. Application Audit
- No `403` status codes are returned by any Nitro server middleware or API handlers.
- All audited endpoints return either `200 OK`, `404 Not Found`, or `462` (custom format error).

### 2. Simulation Results
- **User-Agent Simulation**: Requests using the `bingbot` User-Agent from non-official IPs returned `200 OK`.
- **Official IP Behavior**: Requests from official Bingbot IP ranges (e.g., `52.167.144.166`, `157.55.39.195`) consistently receive `403 Forbidden` on API paths.

### 3. Path Analysis
- **Allowed**: Initial HTML page requests and `/api/log/setError` requests.
- **Blocked**: `/api/getData/...` and `/api/log/setText` requests.

### 4. Infrastructure Verification
- **External VPS Test**: Requests to an external VPS (`http://2.26.114.183/`) from the client resulted in `<no response> Failed to fetch`.
- **Server-Side Entry**: `server/middleware/robots.ts` successfully captures the initial page request from official Bingbot IPs, proving the bot reaches the server for HTML but is blocked for API calls.

## Conclusion
The `403 Forbidden` errors are generated by the infrastructure layer (WAF/Reverse Proxy) before the request reaches the Nitro server. The block is triggered by the combination of official Bingbot IP ranges and API requests.

## Implementation Details

### Client-Side Fingerprinting
Modified [`app/components/helper/HelperDataFetch.vue`](app/components/helper/HelperDataFetch.vue) to log client metadata (User-Agent, Language, Platform, Screen, Referrer, Location) to the server when `bingbot` is detected.

### Server-Side Entry Logging
Modified [`server/middleware/robots.ts`](server/middleware/robots.ts) to log the remote IP and headers of the initial successful page request.

## Recommended Action
Contact the hosting provider to whitelist official Bingbot IP ranges for the `/api/` paths in the WAF/ModSecurity configuration.
